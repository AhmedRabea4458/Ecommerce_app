# E-commerce App

Recently, I worked on an E-Commerce mobile application using Flutter as part of my learning journey in mobile development.

## Overview
This project focuses on building a scalable and structured mobile application using clean architecture principles and proper state management.

The application is designed as a **role-based system**, where the user experience differs depending on whether the user is a **Customer** or a **Seller**.

## Key Features
- User role management (Customer vs Seller)
- Product management (listing, search, categories, details)
- Shopping cart and wishlist functionality
- Role-based navigation flows
- Clean architecture with separation of UI and business logic
- Scalable and maintainable code structure

## Technologies Used
- Flutter
- Firebase (Authentication & Firestore)
- Bloc / Cubit (State Management)

## Purpose
This project helped me:
- Apply Flutter concepts in a real-world scenario
- Understand scalable app architecture
- Improve state management skills
- Build maintainable and extendable applications
